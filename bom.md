# Swift Android Bill of Materials BOM

## Overview

The Android SDK as generated by the "Android SDK build scripts" pull request (https://github.com/swiftlang/swift-docker/pull/467) is a standard Swift cross-compilation SDK as described in [SE-0387](https://github.com/swiftlang/swift-evolution/blob/main/proposals/0387-cross-compilation-destinations.md). The steps for building the SDK — either in a Docker container or directly on the host Linux machine — are described in the [README](https://github.com/swiftlang/swift-docker/blob/e2f696a3ed5f2b3fa0bf83a81c0dcf44be54ad07/swift-ci/sdks/android/README.md) for the PR.

The build products included in the SDK are entirely the outputs of the build script with no additional external files. The build process includes only source projects from the [swiftlang](https://github.com/swiftlang/) and [apple](https://github.com/apple/) organizations, with the following exceptions (as can be seen in the [fetch-sources.sh](https://github.com/swiftlang/swift-docker/blob/e2f696a3ed5f2b3fa0bf83a81c0dcf44be54ad07/swift-ci/sdks/android/scripts/fetch-source.sh) script):

- yams: https://github.com/}jpsim/Yams.git
- libxml2: https://github.com/GNOME/libxml2.git
- curl: https://github.com/curl/curl.git
- BoringSSL: https://boringssl.googlesource.com/boringssl

This follows the precedent of the [static-linux SDK](https://github.com/swiftlang/swift-docker/blob/main/swift-ci/sdks/static-linux/scripts/fetch-source.sh), which also builds each of these projects (with the exception of Yams). 

## External NDK

One difference from other Swift SDKs is that the Swift Android SDK is not entirely self-contained. It relies on the presence of a pre-installed Android Native Development Kit (NDK). The actual dependency is the "sysroot" subset of the NDK (in `toolchains/llvm/prebuilt/darwin-x86_64/sysroot`) that contains the clang cross-compiler for the various supported Android architectures. The contents of the sysroot folder are redistributable under the Apache License 2.0 and BSD-style licenses, but the encompassing NDK itself does require agreeing to a custom license when downloading from https://developer.android.com/ndk/downloads. By separating the NDK sysroot from the rest of the Swift Android SDK, risk of improperly redistributing parts of the NDK are eliminated.

The mechanism by which the Swift Android SDK is connected to the Android NDK is by running a post-install command that links the NDK's sysroot into the Android SDK. This is done with the [scripts/setup-android-sdk.sh](https://github.com/swiftlang/swift-docker/blob/e2f696a3ed5f2b3fa0bf83a81c0dcf44be54ad07/swift-ci/sdks/android/scripts/build.sh#L565C15-L565C32) script that is included in the artifactbundle.
